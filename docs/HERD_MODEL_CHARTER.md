# HERD 모델 헌장

상태: `APPROVED_BASELINE`  
작성일: 2026-07-19  
적용 대상: 차세대 HERD 상태 모델 연구 및 제품 표현

## 1. 결정 요약

HERD는 **가격·추세·시장 참여·위험·기업 상태처럼 장기투자 판단에 필요한 객관적 증거를 결합해, 미국 주식과 시장 ETF의 현재 상태를 0~100으로 표현한 최종 지수**다.

HERD의 목적은 다음과 같다.

1. 서로 다른 장기투자 증거를 하나의 비교 가능한 최종 상태로 압축한다.
2. HERD 기반의 장기 비중 조절로 동일 종목·동일 기간의 무조건적 Buy & Hold보다 높은 비용 차감 후 복리수익을 달성한다.
3. Buy & Hold의 상승을 충분히 포착하면서 최대낙폭과 하방 위험을 줄인다.
4. 특정 종목이나 시대에만 맞지 않는 재현 가능한 장기투자 기준을 만든다.

HERD는 다음을 하지 않는다.

- 다음 날 또는 단기 가격을 맞히는 방향 예측 모델로 사용하지 않는다.
- 사용자의 상황을 반영한 매수·매도 수량을 직접 결정하지 않는다.
- 사용자의 보유 비중이나 투자 성향을 포함하지 않는다.
- 데이터 품질을 시장 상태로 취급하지 않는다.
- 백테스트 수익률 하나만 최대화하도록 최적화하지 않는다.

## 2. 모델 정의

### 2.1 내부 정의

> HERD는 장기투자에 유효하다고 독립적으로 검증된 가격·추세·참여·위험·기업 상태의 증거를 중복 없이 결합한 최종 순서형 상태 지수다.

HERD 자체는 상태를 표현하지만, 성공 여부는 상태 설명의 그럴듯함이 아니라 **사전에 고정된 HERD 기준 전략이 Buy & Hold를 이기는지**로 판정한다.

핵심 단어의 의미:

- **장기**: 기본 판단 기간이 일중·수일이 아니라 3~12개월 이상이다.
- **군중 상태**: 실제 투자자 수를 직접 측정한다는 뜻이 아니다. 가격·거래·기업 데이터에 남은 시장 참여자의 집단 행동과 그 지속 가능성을 추정한다.
- **종목별 역사 비교**: 서로 특성이 다른 종목에 동일 절대 임계값을 강제하지 않는다.
- **순서형 상태 지수**: 64가 32보다 정확히 두 배 밀집됐다는 뜻은 아니다. 점수가 높을수록 상대적으로 밀집된 상태라는 순서를 뜻한다.

### 2.2 외부 제품 정의

> 가격과 시장 데이터에 남은 군중의 밀집과 분산 상태.

제품 화면은 장문의 해설보다 다음 정보를 우선한다.

- 대상 종목 또는 지수
- HERD 점수
- HERD 단계
- 데이터 기준일
- 구성 관측 요소
- 모델 버전과 검증 상태

## 3. 모델 계층

객관적 시장 분석은 HERD 안에서 끝난다. 개인화만 HERD 밖에 둔다.

```text
시점 정합 시장·기업 데이터
    ↓
독립 증거군
  · 군중 위치/참여
  · 추세/상대강도
  · 모멘텀 지속성
  · 하방 위험/취약성
  · 기업 품질/가치/실적(검증 후)
    ↓
중복 제거 · 정규화 · 상태별 결합
    ↓
최종 HERD 0~100 / 5단계
```

```text
최종 HERD
+ 사용자 투자 기준
+ 현재 보유 상태
+ 행동 한도/쿨다운
    ↓
개인 행동 번역
```

현재 v6.1 Action Layer에 있는 추세, 모멘텀, 수명주기, 시장 위험처럼 **사용자와 무관한 객관적 분석 정보는 차세대 HERD 후보 입력**이다. 검증을 통과한 요소는 최종 HERD 계산에 편입하고, 편입 이후 별도의 객관적 Action 점수처럼 이중 계산하지 않는다.

### HERD에 포함 가능한 정보

- 가격 기반 군중 위치
- 장기 추세
- 검증된 모멘텀 지속 또는 반전 특성
- 변동성·낙폭 특성
- 시장 폭과 상대강도
- 시점 복원이 가능한 실적·밸류에이션·자금 흐름

실제 포함 여부는 문헌 근거, 데이터 가능성, 중복성 및 OOS 검증으로 결정한다.

### HERD에 포함하지 않는 정보

- 사용자 보유 수량
- 평균 매수가
- 현금 비중
- 목표 비중
- 투자 기간 설정
- 위험 허용도 설정
- 최근 사용자 행동
- 데이터 품질 점수
- 모델 신뢰도 점수

### 별도 표시할 정보

- `DATA QUALITY`: 입력 데이터의 완전성·최신성
- `MODEL CONFIDENCE`: 해당 상태의 표본 수와 과거 재현성
- `PERSONAL ACTION`: 최종 HERD를 개인 상황에 적용한 비중 범위

`CONFIDENCE`는 방향성 증거가 아니므로 HERD에 가중 합산하지 않는다. 신뢰도가 낮으면 점수를 중립값으로 왜곡하기보다 결과를 보류하거나 개인 행동 폭을 제한한다.

## 4. 대상 사용자와 사용 상황

### 주 사용자

- 미국 주식을 수개월에서 수년 보유하는 개인 장기투자자
- 보유 종목과 관심 종목의 현재 위치를 간단히 확인하려는 사용자
- 급락과 급등에서 감정적인 행동을 줄이고 싶은 사용자

### 핵심 사용 상황

1. 시장 전체 상태 확인: SPY
2. 보유 종목의 상대 상태 확인
3. 신규 진입 전 과도한 밀집 또는 이탈 여부 확인
4. 기존 판단 이후 상태 변화 기록
5. 모델 근거와 검증 한계 확인

### 제외하는 사용 상황

- 초단타와 일중 매매
- 실시간 주문 실행
- 레버리지·옵션 포지션 최적화
- 다음 날 가격 예측
- 자동 종목 추천

## 5. 시간 범위와 갱신 원칙

### 모델 시간 범위

- 주 평가 기간: 6개월, 12개월
- 보조 평가 기간: 1개월, 3개월
- 장기 확인 기간: 24개월

1개월 결과는 조기 이상 탐지에 사용하지만 모델 채택의 주근거로 사용하지 않는다.

### 계산 빈도

- 원천 데이터는 거래일 단위로 갱신할 수 있다.
- HERD는 일별 계산하되 장기 상태 지수답게 과도한 일간 변동을 억제한다.
- 행동 평가는 다음 거래일 시가 체결을 기본으로 한다.
- 동일 상태의 반복 이벤트는 독립 표본처럼 중복 계산하지 않는다.

### 상태 지속성

좋은 HERD는 하루마다 단계가 바뀌지 않아야 한다. 동시에 구조적 변화는 지나치게 늦지 않게 반영해야 한다.

상태 안정성 평가 항목:

- 연간 단계 전환 수
- 전환 후 5거래일 이내 원복 비율
- 점수 일간 절대 변화 중앙값
- 극단 상태 진입 지연
- 급락·급등 구간 반응 지연

## 6. 점수 의미

### 점수 방향

- 낮은 점수: 군중 이탈·분산 방향
- 중간 점수: 균형 방향
- 높은 점수: 군중 밀집 방향

낮은 점수가 자동 매수를 뜻하지 않으며 높은 점수가 자동 매도를 뜻하지 않는다.

### 단계

현재 운영 단계는 연구 기준선으로 유지한다.

| 점수 | 단계 | 상태 |
|---:|---|---|
| 0 ≤ 점수 ≤ 15 | Flee | 군중 이탈 |
| 15 < 점수 ≤ 40 | Scatter | 군중 분산 |
| 40 < 점수 < 60 | Calm | 군중 균형 |
| 60 ≤ 점수 < 75 | Drift | 군중 밀집 시작 |
| 75 ≤ 점수 ≤ 100 | Rush | 군중 밀집 |

차세대 모델이 채택되기 전까지 임계값을 변경하지 않는다. 새 점수 분포가 기존 임계값과 맞지 않으면 점수 캘리브레이션을 먼저 검토하며, 성과가 좋아 보이도록 임계값을 사후 이동하지 않는다.

### 종목 간 비교

HERD 70의 의미가 모든 종목에서 완전히 동일하다고 가정하지 않는다.

비교 가능성은 두 수준으로 평가한다.

1. 종목 내부: 해당 종목의 역사적 상태에서 현재 위치
2. 종목 간: 같은 점수 구간의 향후 결과 분포가 유사한지

종목 간 캘리브레이션이 검증되지 않으면 UI에서 종목 순위를 과도하게 강조하지 않는다.

## 7. 주목표

### 주목표 문장

> HERD 기반 기준 전략은 동일 종목·동일 기간의 무조건적 Buy & Hold보다 높은 비용 차감 후 장기 복리수익을 달성하고, 최대낙폭과 하방 위험도 개선해야 한다.

HERD는 단일 미래 수익률을 맞히는 회귀 모델이 아니다. 최종 판정에서는 다음을 함께 평가한다.

- 비용 차감 후 CAGR과 누적수익률
- Buy & Hold 대비 초과수익률
- 최대낙폭과 하방 변동성
- Calmar·Sortino Ratio
- 상승장 수익 포착률과 하락장 손실 포착률
- 현금 보유 기간과 평균 주식 노출
- 거래 횟수와 회전율
- 원금 회복 기간

### 공식 비교 전략

모델 간 비교를 위해 사용자 설정과 무관한 `HERD Benchmark Strategy`를 둔다.

- 동일한 최초 자본과 평가 기간을 사용한다.
- 다음 거래일 시가에 체결한다.
- 수수료와 슬리피지를 차감한다.
- HERD 구간에 따라 주식·현금 목표 비중만 변경한다.
- 신호가 없는 기간을 임의로 제외하지 않는다.
- 초기 100% 주식, 초기 100% 현금, 정기 적립식 시나리오를 각각 평가한다.

구간별 목표 비중과 쿨다운은 후보 실험 전에 잠그며, Blind holdout 결과를 보고 변경하지 않는다.

### 낮은 점수에 대한 연구 가설

Flee·Scatter는 Calm 대비 다음 중 하나 이상의 특성을 보여야 한다.

- 장기 기대수익 개선
- 향후 초과수익 개선
- 밸류에이션 또는 가격 위치의 회복

단, 추세 훼손과 구조적 부진 때문에 낙폭 위험이 커질 수 있으므로 낮은 점수만으로 행동을 결정하지 않는다.

### 높은 점수에 대한 연구 가설

Drift·Rush는 Calm 대비 다음 중 하나 이상의 특성을 보여야 한다.

- 향후 최대낙폭 증가
- 향후 수익률 비대칭 악화
- 상승 지속 후 반전 위험 증가

강한 장기 추세가 계속되는 `Healthy Rush`가 존재할 수 있으므로 높은 점수만으로 매도를 결정하지 않는다.

## 8. 보조목표

주목표를 훼손하지 않는 범위에서 다음을 추구한다.

1. 설명 가능성: 구성 요소와 계산 상태를 사용자에게 공개할 수 있어야 한다.
2. 안정성: 인접 파라미터와 데이터 구간에서도 결과가 급변하지 않아야 한다.
3. 범용성: 섹터와 종목 특성에 과도하게 종속되지 않아야 한다.
4. 데이터 현실성: 운영 환경에서 지속적으로 수집 가능한 데이터만 사용한다.
5. 시점 정합성: 과거 시점에 알 수 없었던 정보를 사용하지 않는다.
6. 행동 절제: 개인 행동 번역이 불필요하게 잦은 행동을 만들지 않게 한다.

## 9. 비목표

다음 결과가 좋아져도 HERD 자체의 성공으로 간주하지 않는다.

- 특정 소수 종목의 누적수익률 최대화
- 학습 구간 Sharpe Ratio 최대화
- 매수·매도 횟수 증가
- 모든 개별 연도와 모든 단일 시장 구간에서 Buy & Hold 초과
- 하루 또는 일주일 뒤 방향 적중
- 사용자별 포트폴리오 최적화
- 모델이 생성한 자연어 설명의 설득력

## 10. 평가 프레임

### 10.1 상태 분리력

- 단계별 6개월·12개월 결과 분포
- Flee/Scatter와 Calm의 효과 크기
- Drift/Rush와 Calm의 효과 크기
- 점수 십분위별 결과의 순서성
- 부트스트랩 신뢰구간

평균값만 보지 않고 중앙값, 하위 분위수, 분산과 표본 수를 함께 본다.

### 10.2 안정성

- 연도별 효과 방향 일치율
- 섹터별 효과 방향 일치율
- 종목별 효과 방향 일치율
- 파라미터 선택 빈도
- 인접 파라미터 성능 차이
- Fold 간 선택 유지율

### 10.3 일반화

- Walk-forward OOS
- 사전에 잠근 Blind holdout
- 상승·하락·횡보 Regime
- 대형주·중형주
- 기술주·비기술주
- 현존 종목과 과거 편출·합병·상장폐지 종목

### 10.4 운영 가능성

- 데이터 성공 수집률
- 결측 상태 비율
- 계산 시간
- 무료 API 의존성
- 과거 시점 복원 가능성
- 데이터 수정 및 재작성 위험

## 11. 초기 채택 게이트

아래 기준은 차세대 모델 연구 전에 잠그는 초기 게이트다. 연구 결과를 본 후 유리하게 변경하지 않는다. 변경이 필요하면 이유와 변경일을 별도 기록하고 새로운 Blind holdout을 사용한다.

### 필수 게이트

1. Look-ahead bias 검사 통과
2. 발표일 기준 데이터 적용
3. 수수료·슬리피지·다음 거래일 시가 반영
4. Walk-forward OOS와 Blind holdout 분리
5. 모든 후보 실험 이력과 시험 횟수 저장
6. PBO·Deflated Sharpe 및 파라미터 민감도 공개
7. 데이터 품질 실패 시 점수와 신뢰도를 구분
8. v4 대비 결과가 나쁜 종목과 섹터 공개
9. Buy & Hold와 동일한 자본·기간·가격·비용 조건 적용

### 상태 모델 게이트

- 극단 단계와 Calm의 6개월 또는 12개월 결과 분포가 최소 두 개의 독립 평가 구간에서 같은 방향으로 구분될 것
- 전체 결과가 소수 종목에 의존하지 않을 것
- 주요 섹터 다수에서 효과 방향이 유지될 것
- 점수 십분위 결과가 단일 특정 구간에서만 급등하지 않을 것
- 연간 단계 전환 중앙값이 장기 상태 지수의 사용 목적을 해치지 않을 것
- v4 대비 상태 분리력, 안정성, 데이터 현실성 중 최소 두 영역에서 개선되고 나머지 영역에서 중대한 악화가 없을 것

효과 크기와 최소 표본 수의 정확한 수치는 2단계 문헌 조사와 현재 데이터 분포 확인 후, 후보 모델 실험 전에 확정한다.

### 개인 행동 번역 게이트

최종 HERD 채택과 별도로 평가한다.

- 기존 보유자와 신규 진입자 분리
- 6개월·12개월 수익 보존
- MDD 개선
- 행동 횟수
- 행동 이후 반사실 대비 효과
- Healthy Rush 손실 여부

현재 v6.1의 행동 검증 게이트는 과거 연구 기준선으로만 남긴다.

- 수익률 보존율 70% 이상
- MDD 개선 5%p 이상
- 연평균 행동 4~10회

현재 Blind holdout 중앙값은 수익 보존 약 91.1%, MDD 개선 약 2.53%p이므로 운영 채택 기준에 미달한다.

차세대 HERD의 최종 채택에는 위 수익 보존 기준만으로 충분하지 않다. `HERD Benchmark Strategy`가 비용 차감 후 Buy & Hold 초과수익을 내야 하며, 정확한 최소 초과수익·MDD·포착률 기준은 데이터 분포 확인 후 Blind holdout 실행 전에 고정한다.

## 12. 데이터와 연구 원칙

1. 당시 사용할 수 있었던 정보만 사용한다.
2. 수정된 재무정보를 과거에 소급하지 않는다.
3. 현재 지수 구성 종목만으로 과거를 평가하지 않는다.
4. 후보 요소를 합치기 전에 단독 효과와 상관관계를 확인한다.
5. 경제적 설명 없이 성과만 좋은 요소는 보수적으로 제외한다.
6. 무료 데이터로 재현할 수 없는 요소는 운영 모델과 분리한다.
7. 결측값을 0 또는 중립값으로 조용히 대체하지 않는다.
8. 데이터 품질은 시장 점수와 별도로 저장하고 표시한다.

## 13. 버전 정책

### 현재 운영

- `HERD v4`: 운영 상태 점수
- `HERD v6.1 Action Layer`: 객관 요소 통합 실험 및 행동 검증 자료

### 차세대 연구

차세대 HERD는 v4와 v6.1의 객관적 요소를 후보군으로 다시 분해하고, 중복성과 OOS 기여도를 검증한 뒤 새 버전으로 채택한다. v6.1 결과를 이름만 바꿔 승격하지 않는다.

버전 승격 조건:

1. 모델 정의 변경
2. 구성 요소 변경
3. 가중치 또는 결합 방식 변경
4. 점수 캘리브레이션 변경
5. 임계값 변경

각 버전은 다음을 남긴다.

- 모델 카드
- 입력 데이터 목록
- 계산 공식
- 학습·검증 기간
- 후보 시험 횟수
- 채택·기각 이유
- 알려진 한계
