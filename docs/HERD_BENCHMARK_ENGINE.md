# HERD 공정 비교 엔진

상태: `IMPLEMENTED_BASELINE`  
구현일: 2026-07-19  
코드: `data/herd/benchmark_engine.py`

## 목적

HERD 후보와 Buy & Hold가 서로 다른 체결 시점·비용·기간을 사용해 생기는
비교 왜곡을 제거한다. 전략은 목표 주식 비중만 제공하고, 체결과 성과 계산은
공통 엔진이 담당한다.

## 고정 원칙

- 동일한 OHLC 가격 프레임과 평가 구간
- 동일한 최초 자본과 외부 납입
- 신호 발생 후 최소 다음 거래일 시가 체결
- 매수·매도 모두 수수료와 슬리피지 적용
- 미투자 현금의 수익률 명시
- 외부 납입을 투자수익으로 계산하지 않는 time-weighted 일간 수익률
- 최초 매수 비용도 성과에 포함
- 목표 비중 범위를 0~100%로 제한하고 레버리지 금지

## 기본 설정

| 항목 | 기본값 |
|---|---:|
| 최초 자본 | $10,000 |
| 수수료 | 편도 0.10% |
| 슬리피지 | 편도 0.05% |
| 현금 연수익률 | 0% |
| 실행 지연 | 1거래일 |
| 초기 주식 비중 | 100% |

기본값은 비교 엔진의 동작 기준이며 3단계 채택 정책에서 비용 민감도 시나리오를
별도로 잠근다.

## 산출 지표

- 비용 차감 후 누적수익률
- CAGR
- Buy & Hold 대비 초과 CAGR
- 최대낙폭
- 연환산 하방편차
- Sortino Ratio
- Calmar Ratio
- 상승장 포착률
- 하락장 손실 포착률
- 거래 횟수
- 회전율
- 평균 주식 노출
- 평균 현금 비율
- 누적 납입액과 최종 평가액

## 지원 시나리오

- 기존 보유자: 초기 주식 비중 100%
- 신규 진입자: 초기 주식 비중 0%, HERD 목표 비중으로 진입
- 정기 적립식: 날짜별 `contributions` 전달
- 목표 비중 전략: 날짜별 0~1 `target_weights` 전달

모든 신호는 최소 다음 거래일에 체결된다. 같은 날 생성한 종가 기반 신호를
같은 날 시가나 종가로 거래하는 설정은 허용하지 않는다.

## 테스트

`data/tests/test_benchmark_engine.py`에서 다음을 검증한다.

1. 당일 체결 설정 거부
2. 신호 다음 거래일 시가 체결
3. 100% 노출 전략과 Buy & Hold의 완전 동일성
4. 적립금이 수익률로 계산되지 않음
5. 최초 거래 비용이 수익률에 포함됨
6. 필수 위험·포착률 지표 제공

## 남은 연결 작업

이 단계에서는 기존 v4·v6.1 전략 코드를 변경하지 않는다. 3단계에서 채택
정책을 잠근 뒤, 5단계 재평가에서 기존 전략이 목표 비중 시계열을 생성해
이 엔진으로만 평가되도록 어댑터를 연결한다.
