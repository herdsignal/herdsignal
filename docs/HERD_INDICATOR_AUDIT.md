# HERD v4·v6.1 지표 인벤토리 및 중복성 감사

상태: `COMPLETED`  
감사일: 2026-07-19  
코드 기준: `data/config/settings.py`, `data/herd/calculator.py`,
`data/herd/backtest_action_layer.py`, `ActionDecisionService.java`

## 범위

- v4 운영 점수의 원천 지표, 가중치, 기간, 정규화, 데이터 출처
- v4 EPS·섹터 보정의 시점 정합성
- Python 검증 코드와 Java 운영 v6.1의 객관·신뢰도·개인 입력
- 고정 검증 유니버스 55종목, 최근 10년의 가격 지표 Spearman 상관관계

정량 결과 원본은 `data/reports/indicator_audit_v1.json`에 보존한다.

## v4 공식

```text
base =
  monthly RSI percentile      × 0.24
+ weekly RSI percentile       × 0.19
+ 52-week position percentile × 0.19
+ MA200 deviation percentile  × 0.18
+ volume strength percentile  × 0.00
+ weekly MA200 percentile     × 0.20

HERD v4 = clip(base × EPS multiplier × sector multiplier, 0, 100)
```

| 입력 | 기간·공식 | 출처 | 상태 | 감사 판정 |
|---|---|---|---|---|
| 월봉 RSI | 14개월 RSI의 자체 역사 백분위 | yfinance 조정 OHLCV | 24% | 가격·추세군 중복 |
| 주봉 RSI | 14주 RSI의 자체 역사 백분위 | yfinance 조정 OHLCV | 19% | 가격·추세군 중복 |
| 52주 위치 | 252일 고저 범위 내 종가 위치의 백분위 | yfinance 조정 OHLCV | 19% | 매우 높은 중복 |
| MA200 이격 | 종가/200일 SMA 이격의 백분위 | yfinance 조정 OHLCV | 18% | 매우 높은 중복 |
| 거래량 강도 | 5일/20일 평균 거래량 비율의 백분위 | yfinance 조정 거래량 | 0% | 계산하지만 점수 미반영 |
| 200주 MA 위치 | 주봉 종가/200주 SMA의 백분위 | yfinance 조정 OHLCV | 20% | 가격·추세군 중복 |
| EPS 승수 | 최근 2~4분기 연속 beat/miss, 0.85~1.15 | Finnhub | 최신 점수만 | 과거 발표일 부재 |
| 섹터 승수 | 최근 90일 종목-섹터 ETF 상대수익, 0.90~1.10 | Finnhub sector+yfinance | 최신 점수만 | 과거 섹터·승수 복원 미완료 |

### v4 데이터 위험

1. 운영 수집은 `auto_adjust=True`라 분할·배당 조정 가격을 사용한다.
2. EPS 응답에는 신뢰할 발표일이 없어 과거 점수에는 적용할 수 없다.
3. 기존 v4 백테스트는 **현재 EPS·섹터 승수를 과거 3년 전체에 동일 적용한 sanity check**다. 역사 성과 증거가 아니다.
4. 과거 HERD 백필은 EPS·섹터 승수를 1.0으로 두므로 최신 운영 v4와 역사 차트의 공식이 다르다.
5. 상장 기간이 200주보다 짧으면 `ma200_weekly`가 가용 기간으로 자동 축소돼 종목 간 정의가 달라진다.

## v6.1 인벤토리

### 객관적 입력

| 입력 | 계산 | 기간 | 중복 판정 |
|---|---|---|---|
| 안정화 HERD | 경계 ±2pt 재교차 시 이전 구간 유지 | 직전 점수 | v4 파생 |
| 추세 품질 | 200주 MA, MA200 이격, 52주 위치, 섹터 승수, EPS 승수 규칙 점수 | 혼합 | v4 입력 5개 재사용 |
| HERD 모멘텀 | 5일·20일 HERD 변화와 가속도 | 5/20일 | v4의 2차 파생 |
| Lifecycle | 동일 signal 지속 기간 | 1~45+일 | v4 단계 파생 |
| Healthy/Exhausted 상태 | HERD+추세+모멘텀 조건 | 혼합 | 위 파생값 재결합 |

Python 백테스트의 `trend_quality`는 종가>MA200, MA50>MA200,
MA200 63일 기울기, MA200 이격, 52주 위치로 구성된다. Java 운영 로직은
200주 MA, MA200 이격, 52주 위치, 섹터·EPS 승수를 사용한다. 이름은 같지만
공식이 서로 다르므로 현재 v6.1 백테스트와 운영 응답은 완전히 동일한 모델이 아니다.

## 55종목 상관관계 결과

기간은 2016-07-18~2026-07-17, 유효 결합 관측치는 80,135개다.
각 시점까지 존재한 가격만 사용해 자체 역사 백분위를 계산했다.

| 지표 쌍 | Spearman | 판정 |
|---|---:|---|
| 52주 위치 ↔ MA200 이격 | 0.9018 | 치명적 중복 |
| 주봉 RSI ↔ MA200 이격 | 0.8983 | 치명적 중복 |
| 주봉 RSI ↔ 52주 위치 | 0.8545 | 치명적 중복 |
| MA200 이격 ↔ 200주 MA 위치 | 0.7730 | 높은 중복 |
| 주봉 RSI ↔ 200주 MA 위치 | 0.7506 | 높은 중복 |
| 52주 위치 ↔ 200주 MA 위치 | 0.7391 | 높은 중복 |
| 월봉 RSI ↔ 200주 MA 위치 | 0.7239 | 높은 중복 |

현재 활성 가중치 100%가 모두 가격·추세 계열이고, 그중 76%는 서로
0.85 이상의 상관관계를 보이는 세 지표군과 직접 연결된다. 따라서 현재
HERD는 여러 독립 증거의 합이라기보다 **유사한 가격 상태를 여러 시간축으로
반복 투표한 점수**에 가깝다.

거래량 지표는 다른 가격 지표와 중복이 낮지만 가중치가 0이라 실제 HERD에는
참여 정보가 없다.

## 결정

1. v4는 차세대 모델의 `B0` 기준선으로만 보존한다.
2. 주봉 RSI·52주 위치·MA200 이격을 독립 가중치로 동시에 유지하지 않는다.
3. 가격·추세군은 다음 단계에서 대표값 하나 또는 통계적으로 축약된 값으로 만든다.
4. 월봉 RSI와 200주 MA도 단독 OOS 기여가 확인되기 전에는 유지하지 않는다.
5. EPS는 발표일·당시 컨센서스가 복원될 때까지 역사 모델에서 제외한다.
6. 섹터 상대강도는 point-in-time 시계열로 다시 계산한다.
7. Python과 Java v6.1 공식을 하나의 모델 명세로 통일하기 전에는 운영 채택 근거로 사용하지 않는다.
8. 데이터 품질·개인 조건·쿨다운은 최종 HERD 방향 점수에 넣지 않는다.

## 재현

```bash
cd data
.venv/bin/python herd/indicator_inventory.py \
  --period 10y \
  --output reports/indicator_audit_v1.json
```

`--tickers`를 생략하면 고정 55종목 검증 유니버스를 사용한다. 중복 후보
기본 임계값은 절대 Spearman 0.70이며 `--threshold`로 변경할 수 있다.

## 제한사항

- 주·월 RSI 감사 시계열은 마감된 주·월 캔들을 다음 거래일부터 사용한다.
  운영 최신 계산은 진행 중인 주·월 캔들을 포함하므로 값이 일부 다를 수 있다.
- 200주 MA 상관관계는 완전한 200주 이력이 생긴 뒤부터 계산했다.
  운영 코드의 단기 상장 종목 가변 기간 폴백은 정량 결과에서 제외했다.
- 상관관계는 중복을 보여줄 뿐 제거할 지표를 결정하지 않는다.
  최종 선택은 단독 OOS 기여와 ablation test로 수행한다.
